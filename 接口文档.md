# 接口文档

## 基础信息

- **Base URL**：`http://1.14.75.15`

- **所有接口前缀**：`/api`

- **请求头**：`Content-Type: application/json`（文件上传接口使用 `multipart/form-data`）

- **图片访问**：接口返回的图片路径（如 `avatarUrl`、`file_path`、`preview_path`）均为相对路径，需拼接 Base URL 获得完整可访问地址。
  例如：`http://1.14.75.15/uploads/avatar/avatar-1780926024636-358172155.jpg`

  如果返回的图片路径为空的话，可以用下面默认的两个路径：

  `http://1.14.75.15/uploads/avatar/default_avatar.jpg`

  `http://1.14.75.15/uploads/observation_photos/default_photo.jpg`



## 通用响应格式

所有接口返回 JSON 格式，包含以下字段：

| 字段    | 类型    | 说明                             |
| :------ | :------ | :------------------------------- |
| success | boolean | 请求是否成功                     |
| data    | object  | 成功时返回的数据，格式见各接口   |
| message | string  | 失败时的错误描述（成功时可能无） |

**成功示例**：

```json
{ "success": true, "data": { ... } }
```



**失败示例**：

```json
{ "success": false, "message": "用户名或密码错误" }
```





## 数据模型（对象结构）

所有接口返回的数据都基于以下模型。

### User（用户）

| 字段名      | 类型   | 说明                                       |
| :---------- | :----- | :----------------------------------------- |
| userId      | number | 用户ID                                     |
| username    | string | 管理员用户名，微信用户为 null              |
| email       | string | 管理员邮箱，微信用户为 null                |
| role        | string | observer / reviewer / admin                |
| status      | string | active（正常） / banned（封禁）            |
| nickname    | string | 微信用户昵称                               |
| avatarUrl   | string | 头像相对路径，如 `/uploads/avatar/xxx.jpg` |
| createdAt   | string | 注册时间（ISO 8601 UTC）                   |
| lastLoginAt | string | 最后登录时间，未登录过则为 null            |

**JSON 示例**：

```json
{
  "userId": 2,
  "username": null,
  "email": null,
  "role": "observer",
  "status": "active",
  "nickname": "张三",
  "avatarUrl": "/uploads/avatar/avatar-xxx.jpg",
  "createdAt": "2025-01-01T00:00:00.000Z",
  "lastLoginAt": "2026-06-08T10:30:00.000Z"
}
```



### Location（地点）

| 字段名      | 类型   | 说明                           |
| :---------- | :----- | :----------------------------- |
| locationId  | number | 地点ID                         |
| name        | string | 地点名称                       |
| latitude    | number | 纬度（-90 到 90），可为 null   |
| longitude   | number | 经度（-180 到 180），可为 null |
| description | string | 描述，可为 null                |

### Species（物种）

| 字段名      | 类型   | 说明                     |
| :---------- | :----- | :----------------------- |
| speciesId   | number | 物种ID                   |
| speciesName | string | 物种名称                 |
| description | string | 描述，可为 null          |
| createdAt   | string | 创建时间（ISO 8601 UTC） |

### Photo（照片）

| 字段名      | 类型   | 说明                                                         |
| :---------- | :----- | :----------------------------------------------------------- |
| photoId     | number | 照片ID                                                       |
| filePath    | string | 原图相对路径，如 `/uploads/observation_photos/obsPhoto-xxx.jpg` |
| previewPath | string | 预览图相对路径，如 `/uploads/observation_photos/preview_xxx.jpg` |

| obsId        | number              | 观测ID                                                       |

| reviewedAt   | string or null      | 审核时间                                                     |
| identifiedAt | string or null      | 鉴定时间                                                     |
| user         | User 对象           | 观测者                                                       |
| species      | Species 对象或 null | 关联的物种                                                   |
| location     | Location 对象       | 观测地点                                                     |

## 接口详细说明

### 一、用户模块 `/api/users`

#### 1. 创建微信用户（注册）

- **接口**：`POST /api/users/wechat`
- **完整网址示例**：`http://1.14.75.15/api/users/wechat`
- **请求方式**：`multipart/form-data`
- **请求参数**：
  - `openid`（必填，字符串）：微信 openid
  - `nickname`（可选，字符串）：用户昵称
  - `avatar`（可选，文件）：头像图片（支持 jpg/png 等，自动压缩为宽度 500px 的 jpg）
- **返回说明**：成功时 `data` 为 `User` 对象。如果 openid 已存在，返回 `success: false` 且 `message` 为“该微信用户已存在”。

#### 2. 创建管理员用户（注册）

- **接口**：`POST /api/users/admin`
- **完整网址示例**：`http://1.14.75.15/api/users/admin`
- **请求方式**：`multipart/form-data`
- **请求参数**：
  - `username`（必填，字符串）：用户名，必须唯一
  - `email`（必填，字符串）：邮箱，必须唯一
  - `password`（必填，字符串）：密码（明文，后端会加密存储）
  - `role`（可选，字符串，默认 `admin`）：`observer` / `reviewer` / `admin`
  - `avatar`（可选，文件）：头像图片
- **返回说明**：成功时 `data` 为 `User` 对象。用户名或邮箱已存在时返回错误。

#### 3. 用户登录（统一接口）

- **接口**：`POST /api/users/login`

- **完整网址示例**：`http://1.14.75.15/api/users/login`

- **请求方式**：`application/json`

- **请求参数**（二选一）：

  **微信用户登录**（无需密码）：

  ```json
  { "openid": "oxxx123456" }
  ```

  

  **管理员登录**：

  ```json
  { "username": "admin", "password": "123456" }
  ```

  

- **返回说明**：成功时 `data` 为 `User` 对象，且 `lastLoginAt` 字段会自动更新为当前服务器时间。如果微信用户尚未注册，返回错误“该微信用户不存在，请先注册”。如果账号被封禁，返回错误“账号已被封禁”。

#### 4. 按 ID 获取用户信息

- **接口**：`GET /api/users/:id`
- **完整网址示例**：`http://1.14.75.15/api/users/1`
- **请求方式**：`GET`
- **说明**：仅查询用户信息，不更新最后登录时间。

#### 5. 按 openid 获取用户信息

- **接口**：`GET /api/users?openid=xxx`
- **完整网址示例**：`http://1.14.75.15/api/users?openid=oxxx123456`
- **请求方式**：`GET`
- **说明**：仅查询用户信息，不更新最后登录时间。推荐使用登录接口进行身份验证。

#### 6. 更新微信用户资料（昵称/头像URL）

- **接口**：`PUT /api/users/:id/profile`

- **完整网址示例**：`http://1.14.75.15/api/users/2/profile`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "nickname": "新昵称", "avatarUrl": "/new/path.jpg" }
  ```

  

  至少提供一个字段。

- **返回说明**：成功时返回更新后的 `User` 对象。

#### 7. 更新管理员资料（用户名/邮箱/密码）

- **接口**：`PUT /api/users/:id/admin`

- **完整网址示例**：`http://1.14.75.15/api/users/1/admin`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "username": "新用户名", "email": "新邮箱", "password": "新密码" }
  ```

  

  所有字段可选，但至少提供一个。

- **返回说明**：成功时返回更新后的 `User` 对象。

#### 8. 上传/更新头像

- **接口**：`POST /api/users/:id/avatar`
- **完整网址示例**：`http://1.14.75.15/api/users/2/avatar`
- **请求方式**：`multipart/form-data`
- **请求参数**：`avatar`（文件，必填）
- **说明**：新头像会自动压缩为宽度 500px 的 jpg，并删除旧头像文件（非默认头像）。
- **返回说明**：成功时返回更新后的 `User` 对象。

#### 9. 修改用户状态（封禁/解封）

- **接口**：`PATCH /api/users/:id/status`

- **完整网址示例**：`http://1.14.75.15/api/users/2/status`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "status": "banned" }   // 或 "active"
  ```

  

- **返回说明**：成功时返回更新后的 `User` 对象。

#### 10. 修改用户角色

- **接口**：`PATCH /api/users/:id/role`

- **完整网址示例**：`http://1.14.75.15/api/users/2/role`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "role": "reviewer" }   // observer / reviewer / admin
  ```

  

- **返回说明**：成功时返回更新后的 `User` 对象。

#### 11. 删除用户

- **接口**：`DELETE /api/users/:id`
- **完整网址示例**：`http://1.14.75.15/api/users/2`
- **请求方式**：`DELETE`
- **说明**：会同时删除该用户的头像文件（如果不是默认头像）。
- **返回说明**：成功时 `data` 无内容，`message` 为“用户已删除”。

#### 12. 获取用户列表（分页+筛选）

- **接口**：`GET /api/users`

- **完整网址示例**：`http://1.14.75.15/api/users?page=2&pageSize=10&role=observer&status=active&keyword=张`

- **请求参数**（query string）：

  - `page`：页码，默认 1
  - `pageSize`：每页数量，默认 20
  - `role`：筛选角色（observer/reviewer/admin）
  - `status`：筛选状态（active/banned）
  - `keyword`：模糊搜索用户名、昵称、邮箱

- **返回格式**：

  ```json
  {
    "success": true,
    "data": {
      "list": [ ...User对象数组 ],
      "total": 100,
      "page": 2,
      "pageSize": 10
    }
  }
  ```

  

#### 13. 唯一性检查接口

这些接口用于前端实时校验用户名/邮箱/openid 是否可用。

- **检查用户名**：`GET /api/users/check-username?username=xxx`
  - 完整网址：`http://1.14.75.15/api/users/check-username?username=admin`
  - 返回：`{ "success": true, "data": { "username": "admin", "available": false } }`
- **检查邮箱**：`GET /api/users/check-email?email=xxx`
  - 完整网址：`http://1.14.75.15/api/users/check-email?email=admin@example.com`
  - 返回：`{ "success": true, "data": { "email": "admin@example.com", "available": false } }`
- **检查 openid**：`GET /api/users/check-openid?openid=xxx`
  - 完整网址：`http://1.14.75.15/api/users/check-openid?openid=oxxx123`
  - 返回：`{ "success": true, "data": { "openid": "oxxx123", "available": true } }`

### 二、地点模块 `/api/locations`

#### 1. 创建地点

- **接口**：`POST /api/locations`

- **完整网址**：`http://1.14.75.15/api/locations`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  {
    "name": "西湖",
    "latitude": 30.25,
    "longitude": 120.15,
    "description": "杭州著名景点"
  ```

  经纬度和描述可选。

- **返回**：`data` 为 `Location` 对象。

- **完整网址**：`http://1.14.75.15/api/locations/1`

- **接口**：`PUT /api/locations/:id`

- **请求参数**：支持 `name`, `latitude`, `longitude`, `description` 中的任意字段。
- **返回**：更新后的 `Location` 对象。

#### 4. 删除地点

- **接口**：`DELETE /api/locations/:id`
- **完整网址**：`http://1.14.75.15/api/locations/1`
- **返回**：`{ "success": true, "message": "地点已删除" }`

- **完整网址**：`http://1.14.75.15/api/locations?page=1&pageSize=10&keyword=西湖`

- **请求参数**：`page`, `pageSize`, `keyword`（模糊搜索名称或描述）

- **返回**：

  ```json
  {
    "success": true,
    "data": {
      "list": [ Location对象数组 ],
      "total": 50,
      "page": 1,
      "pageSize": 10

  }
  ```

### 四、观测记录模块 `/api/observations`

#### 1. 创建观测记录（可上传照片）

- **接口**：`POST /api/observations`
- **完整网址**：`http://1.14.75.15/api/observations`
- **请求方式**：`multipart/form-data`
- **表单字段**：
  - `userId`（必填，数字）
  - `locationId`（必填，数字）
  - `speciesId`（可选，数字）
  - `content`（可选，字符串）
  - `status`（可选，字符串，默认 `pending_review`）
  - `photos`（可选，文件数组，字段名 `photos`，最多 10 个文件）
- **返回**：成功时 `data` 为完整的 `Observation` 对象（包含照片数组）。

#### 2. 获取观测记录详情

- **接口**：`GET /api/observations/:obsId`
- **完整网址**：`http://1.14.75.15/api/observations/1`
- **返回**：`data` 为 `Observation` 对象。

#### 3. 更新观测记录（基本信息）

- **接口**：`PUT /api/observations/:obsId`

- **完整网址**：`http://1.14.75.15/api/observations/1`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "userId": 1, "speciesId": 2, "locationId": 3, "content": "新内容" }
  ```

  

  所有字段可选，但 `userId` 必须提供用于权限验证（只有观测者本人可修改）。

- **返回**：更新后的 `Observation` 对象。

#### 4. 更新观测记录状态

- **接口**：`PATCH /api/observations/:obsId/status`

- **完整网址**：`http://1.14.75.15/api/observations/1/status`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "userId": 1, "status": "approved" }
  ```

  

  有效状态：`pending_review`, `approved`, `rejected`, `needs_identification`, `identified`

- **说明**：`userId` 用于记录操作者，暂不强制权限。

- **返回**：更新后的 `Observation` 对象。

#### 5. 删除观测记录（级联删除照片文件）

- **接口**：`DELETE /api/observations/:obsId`
- **完整网址**：`http://1.14.75.15/api/observations/1`
- **请求方式**：`application/json`
- **请求参数**：`{ "userId": 1 }`（只有观测者本人可删除）
- **返回**：`{ "success": true, "message": "观测记录已删除" }`

#### 6. 追加照片到已有观测

- **接口**：`POST /api/observations/:obsId/photos`
- **完整网址**：`http://1.14.75.15/api/observations/1/photos`
- **请求方式**：`multipart/form-data`
- **表单字段**：
  - `userId`（必填，文本字段）
  - `photos`（必填，文件数组，最多 10 个）
- **返回**：更新后的完整 `Observation` 对象（包含所有照片）。

#### 7. 删除观测中的某张照片

- **接口**：`DELETE /api/observations/:obsId/photos/:photoId`
- **完整网址**：`http://1.14.75.15/api/observations/1/photos/5`
- **请求方式**：`application/json`
- **请求参数**：`{ "userId": 1 }`
- **返回**：`{ "success": true, "message": "照片已删除" }`

#### 8. 观测记录列表（分页+筛选）

- **接口**：`GET /api/observations`
- **完整网址**：`http://1.14.75.15/api/observations?page=1&pageSize=10&userId=2&speciesId=3&locationId=4&status=pending_review&keyword=银杏`
- **请求参数**（全部可选）：
  - `page`, `pageSize`
  - `userId`, `speciesId`, `locationId`
  - `status`

### 五、鉴定请求模块 `/api/identification-requests`

#### 1. 创建鉴定请求

- **接口**：`POST /api/identification-requests`

- **完整网址**：`http://1.14.75.15/api/identification-requests`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  { "obsId": 1, "reqSpeciesName": "银杏", "reviewNote": "请专家鉴定" }
  ```

  

  `reqSpeciesName` 和 `reviewNote` 可选。

- **说明**：一个观测只能有一个未完成的鉴定请求。

- **返回**：`data` 为 `IdentificationRequest` 对象（包含完整的观测、物种等嵌套对象）。

#### 2. 获取鉴定请求详情

- **接口**：`GET /api/identification-requests/:reqId`
- **完整网址**：`http://1.14.75.15/api/identification-requests/1`
- **返回**：`data` 为 `IdentificationRequest` 对象。

#### 3. 更新鉴定请求（执行鉴定或拒绝）

- **接口**：`PUT /api/identification-requests/:reqId`

- **完整网址**：`http://1.14.75.15/api/identification-requests/1`

- **请求方式**：`application/json`

- **请求参数**：

  ```json
  {
    "status": "identified",
    "resultSpeciesId": 2,
    "reviewNote": "确认为银杏",
    "reviewerId": 10
  }
  ```

  

  支持部分字段：

  - `status`：`identified` 或 `rejected`
  - `reqSpeciesName`：修改推测物种名
  - `resultSpeciesId`：鉴定结果物种 ID
  - `reviewNote`：鉴定备注
  - `reviewerId`：鉴定人 ID

- **特殊逻辑**：当 `status` 变为 `identified` 且提供了 `resultSpeciesId` 时，系统会自动更新关联观测记录的 `species_id` 和状态为 `identified`，并设置鉴定时间。

- **返回**：更新后的 `IdentificationRequest` 对象。

#### 4. 删除鉴定请求（硬删除）

- **接口**：`DELETE /api/identification-requests/:reqId`
- **完整网址**：`http://1.14.75.15/api/identification-requests/1`
- **返回**：`{ "success": true, "message": "鉴定请求已删除" }`

#### 5. 鉴定请求列表（分页+筛选）

- **接口**：`GET /api/identification-requests`
- **完整网址**：`http://1.14.75.15/api/identification-requests?page=1&pageSize=10&status=pending&obsId=1&reviewerId=10&keyword=银杏`
- **请求参数**：
  - `page`, `pageSize`
  - `status`：pending / identified / rejected
  - `obsId`：观测记录ID
  - `reviewerId`：鉴定人ID
  - `keyword`：搜索 `req_species_name`
- **返回**：分页结构，`list` 中为 `IdentificationRequest` 对象。
