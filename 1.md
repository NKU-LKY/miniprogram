**不能单靠前端把 mock openid 换成「微信生成的 openid」**。这是微信的安全设计，不是项目写法问题。

## 为什么前端拿不到真实 openid

小程序里 `wx.login()` 只能拿到 **`code`**（临时凭证，约 5 分钟有效），**不会**直接返回 openid。

真实 openid 必须用 **`code` + 小程序 AppSecret** 在后端调用微信 `auth.code2Session` 才能换到。AppSecret 不能放在前端，否则等于泄露密钥。

所以：

| 方式 | 同一微信号跨设备是否一致 |
|------|------------------------|
| 当前 mock openid（本地 storage） | ❌ 每台设备不同 |
| 真实 openid（后端 code2Session） | ✅ 同一小程序下始终相同 |

---

## 正确做法（需要前后端一起改）

### 1. 后端新增接口（接口文档里目前还没有）

例如：

```
POST /api/auth/wechat
Body: { "code": "wx.login 返回的 code" }
Response: { "success": true, "data": { "openid": "oXXXX..." } }
```

后端内部：`code` → 调微信 `code2Session` → 返回 `openid`（也可在这一步直接做登录/注册）。

我这边探测过 `http://1.14.75.15/api/auth/wechat`，目前返回 **Cannot POST**，说明**后端还没提供这个能力**。现有接口仍要求前端自己传 `openid`：

```217:248:接口文档.md
  - `openid`（必填，字符串）：微信 openid
  ...
  ```json
  { "openid": "oxxx123456" }
  ```
```

### 2. 前端改成传 `code`，不再自己造 openid

把现在的 `getOrCreateMockOpenid()` 换成类似：

```typescript
// 示意：真实登录流程
async function getWechatOpenid(): Promise<string> {
  const { code } = await new Promise<{ code: string }>((resolve, reject) => {
    wx.login({ success: resolve, fail: reject })
  })

  // 必须走后端换 openid
  const { openid } = await request<{ openid: string }>('/api/auth/wechat', {
    method: 'POST',
    data: { code },
  })

  return openid  // 可选：缓存到 storage，减少重复请求
}
```

注册、登录仍调用现有 `/api/users/wechat`、`/api/users/login`，只是 `openid` 来自后端兑换，而不是 `wx_mock_xxx`。

## 需要注意的一点

电脑上已用 **mock openid** 注册的账号，**不会**自动变成真实 openid 那条。换真实 openid 后：

- 同一微信号在手机上会识别为「新用户」（需重新注册），或  
- 后端做数据迁移，把 mock 记录合并/删除  

这是数据问题，不是前端改一行就能解决的。

## 结论

| 问题 | 答案 |
|------|------|
| 只改前端、不传 mock、直接用微信 openid？ | **不行**，前端无法生成 openid |
| 同一微信号电脑/手机同步？ | **可以**，但要后端提供 `code → openid`，前端传 `code` |
